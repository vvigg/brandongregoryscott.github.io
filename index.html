Geometric Dealies

March 2016

This was the first assignment for CSC 220: Multimedia Programming, where we were given handout code that drew combinations of primitive 2D shapes moving around the screen. Our objective was to convert the sketch to use 3D shapes instead, and we were mostly given creative freedom aside from having at least 2 shapes to form a dealy, supporting key presses to activate different effects: 'f' to freeze the canvas, 'd' to activate a demo mode, and 'c' to change camera perspectives.

I added commands 'e' to completely erase the canvas, 'r' to reinitialize the starting dealies (3), and 's' to spawn a new dealy. It's a pretty simple program, but it was fun to create, and it was a nice introduction to P5js (and graphical programming in general.)

Memory Matching

Spring 2016

This was the semester-long project for CSC 421: Web-based Software Development. The ultimate objective was a web-based memory card matching game that could support up to 2 players, either human or computers. The backend was written in plain Java, while the frontend was handled with the Google Web Toolkit Framework which allows developers to write Java code that is later compiled into Javascript.

This project was especially fun for a few reasons: It was my first attempt at actually designing a GUI, which was quite the challenge, but also a good learning experience. I was allowed (and even encouraged) to dedicate a theme to the game, and the obvious choice was to base it around Yu-Gi Oh, one of my favorite childhood shows. I used images of the actual trading cards and sound effects from the show to create a truly unique experience.

MIDI Controller

April 2016

This project was the final assignment for CSC 220: Multimedia Programming. The objective was to create a custom instrument for a user to play with the keyboard, using different alphabetic keys to map to pitch frequencies that are sent in MIDI messages. The requirements were to map the alphabetical keys, lower and uppercase, to pitch ranges, and to give the user some way of changing the patch (instrument value), channel (track), and volume level of the MIDI message.

Aside from that, we were given a huge amount of freedom to design the program how we wanted, both visually and functionally. I decided to provide an interface of "NoteBox" objects that each had their own patch, pitch, channel, and volume values. These NoteBoxes could be activated temporarily by holding the space bar, or played indefinitely (if the instrument supports so) with the enter key. The variables could be entered into a global buffer in the top left corner of the screen by holding modifier keys (tab, shift, option, command) and either alphabetic keys (for pitch) or numeric keys (for all other values). These global values could be copied from or pasted into a NoteBox with control-c or control-v respectively.

I had a lot of fun working with this project because I had a clear plan for what I wanted to do with it as soon as it was handed out - and I got to play with audio processing for the first time, which was a great experience.

PShapes

March 2016

This was the third assignment for CSC 220: Multimedia Programming. The objective was to create a custom PShape object, which was is a custom grouping of shapes that are constructed from given vertices. We were required to use an image to texture the PShape object and to tint each object on screen differently.

This project was neat because I added a lot of randomness to it - the tint colors, stroke weights, rotations, and movements are randomized. They are bounded, of course, so two of the objects will move up and down while the other two move left and right, but it creates a 'screensaver' type effect if you let it sit for a while, which I think turned out really cool.

MoodMirror

Spring 2016

This was the collaboration project for CSC 220: Multimedia Programming. Our class broke up into teams that collaborated with Communication Design students to create unique, interactive projects to display at the end of the semester. I worked with Brendan Lilly, another CSC major, and Liv Carberry, a CD major, throughout the duration of the semester to bring this project to life.

The project is an interactive installation that displays graphics based on the user's mood. The mood data was retrieved from Sightcorp's FACE API which we called from our P5js sketch. The sketch would take a photo from the webcam every 1 second, send it to the FACE API, and then keep a running log of the mood data for the user.

Liv contributed the low-poly masks and gifs that are displayed in the sketch, while Brendan and I wrote the P5js code. The project installation will be displayed on a monitor with 2-way acrylic over top, to preserve visiblity of the user's face and still display the graphics from the monitor.

This was a great experience because it was the first time Brendan or I had done a real group project in our discipline, let alone a cross-discipline collaboration, and it turned out well.
